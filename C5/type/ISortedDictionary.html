Cut(IComparable cutFunction, KeyValuePair& lowEntry, Boolean& lowIsValid, KeyValuePair& highEntry, Boolean& highIsValid)

Given a "cut" function from the items of the sorted collection to int whose only sign changes when going through items in increasing order can be

The "cut" function is supplied as the CompareTo method of an object c implementing IComparable. A typical example is the case where K is comparable and c is itself of type K.

This method performs a search in the sorted collection for the ranges in which the "cut" function is negative, zero respectively positive. If K is comparable and c is of type K, this is a safe way (no exceptions thrown) to find predecessor and successor of c.

If the supplied cut function does not satisfy the sign-change condition, the result of this call is undefined.

public abstract Boolean Cut(IComparable cutFunction, KeyValuePair& lowEntry, Boolean& lowIsValid, KeyValuePair& highEntry, Boolean& highIsValid)

Parameters

cutFunction

IComparable

The cut function K to int, given by the CompareTo method of an object implementing IComparable.

lowEntry

Returns the largest item in the collection, where the cut function is positive (if any).

lowIsValid

Boolean&

Returns true if the cut function is positive somewhere on this collection.

highEntry

KeyValuePair&

Returns the least item in the collection, where the cut function is negative (if any).

highIsValid

Boolean&

Returns
